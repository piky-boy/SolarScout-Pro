# SolarScout Pro — Project Context

## Overview
SolarScout Pro is an automated solar-lead-generation SaaS for commercial / industrial
rooftops across four European markets: **Romania, Spain, Portugal, Albania**.

Users enter a city / region, the app scans the area for warehouses, factories,
commercial and retail buildings, estimates each roof's area, and saves the
results as leads (per-user). Leads can be filtered, searched, sorted,
inspected on a satellite map and exported to CSV / Excel.

## Tech Stack
- **Next.js 14** (App Router) + TypeScript
- **Prisma 6** + **PostgreSQL** (Abacus-hosted DB)
- **NextAuth v4** with Prisma adapter — supports both **Google SSO** and
  email/password (CredentialsProvider).
  - Google SSO uses **placeholder** Client ID/Secret (`GOOGLE_CLIENT_ID` /
    `GOOGLE_CLIENT_SECRET` in `.env`) until the user provides real values.
- **Mapbox GL JS 3** for satellite map rendering + forward geocoding.
- **Overpass API (OpenStreetMap)** for free, key-less commercial-building
  detection (tags: `building=warehouse|factory|industrial|commercial|retail|...`).
  - Residential tags (`house`, `apartments`, etc.) are filtered out; minimum
    roof area of 200 m² further removes residential noise.
- **xlsx** for Excel exports (CSV built manually).
- **Tailwind CSS 3** + existing `components/ui/*` primitives, `lucide-react`
  icons, `framer-motion` animations, `sonner` toasts.

## Visual Style
- **Brand accent:** amber → orange → rose gradient (`amber-500`, `orange-500`,
  `rose-500`). Use `text-amber-500`, `bg-amber-500` as the primary accent.
- **Logo:** sun icon on gradient amber→orange square + wordmark
  `Solar[Scout]amber Pro`.
- **Typography:** `font-display` (Plus Jakarta Sans) for headings, `font-sans`
  (DM Sans) for body, `font-mono` (JetBrains Mono) for numbers.
- **Max content width:** 1200 px; site header is sticky & glassy.

## Key Files
- `prisma/schema.prisma` — Users, Accounts, Sessions, VerificationToken, Lead,
  SearchHistory. `Lead` has unique `(userId, osmId)` to prevent duplicates.
- `lib/auth.ts` — NextAuth options (Google + Credentials, JWT strategy, redirect
  callback, explicit cookie config).
- `lib/mapbox.ts` — forward geocoding restricted to `country=ro,es,pt,al`.
- `lib/overpass.ts` — Overpass query + classification + roof-area estimation.
- `lib/geometry.ts` — shoelace-based polygon area in m² (equirectangular local
  projection).
- `app/api/leads/generate/route.ts` — end-to-end lead generation pipeline.
- `app/api/leads/export/route.ts` — CSV / xlsx export (by ids or by filters).
- `app/dashboard/_components/dashboard-client.tsx` — main scouting UI.
- `app/leads/_components/leads-client.tsx` — table with filters / sort /
  bulk select / export / delete.

## Pages
- `/` — marketing landing (hero with live Mapbox static-tile preview, features,
  how-it-works, CTA, footer).
- `/login` — email+password + Google SSO (Google button is disabled with a
  warning banner until real credentials are provided).
- `/dashboard` — greeting, quick stats (animated counters), search w/
  autocomplete → generates leads → renders them on Mapbox satellite map with
  clickable markers, side panel with selected-lead details, recent searches,
  top rooftops preview.
- `/leads` — full leads table with search, country/city/type selects, roof-area
  range, sortable columns, bulk select, CSV/Excel export, delete.
- `/leads/[id]` — detail page with zoomed satellite map, stats grid, editable
  notes + status.

## Important Patterns
- API routes use `export const dynamic = 'force-dynamic'` and `getServerSession(authOptions)`
  to gate requests by `session.user.id`.
- Every Mapbox component is client-only, dynamically imported with `ssr:false`.
- Mapbox static-tile URLs on the landing page are assembled via helper
  function to avoid CDN URL rewriters mangling the hardcoded URL.
- Seed script creates `john@doe.com / johndoe123` (ADMIN) + six realistic sample
  leads across all four supported countries for demo purposes.

## Env Vars
- `DATABASE_URL`, `NEXTAUTH_SECRET` — set by framework.
- `NEXT_PUBLIC_MAPBOX_TOKEN` + `MAPBOX_TOKEN` — from the configured Mapbox secret.
- `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` — placeholders, user must supply
  real values for Google SSO to work.
