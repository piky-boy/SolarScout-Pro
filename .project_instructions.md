# SolarScout Pro — Project Memory

## Overview
SolarScout Pro is a solar lead-generation SaaS for commercial rooftop prospects in **Romania, Spain, Portugal, and Albania**. It combines satellite building detection with a lead pipeline and rooftop solar potential analysis.

- **Project root**: `/home/ubuntu/solarscout_pro`
- **Production URL**: https://solarscout-pro-7foyl3.abacusai.app
- **Tech**: Next.js 14 App Router · TypeScript · Prisma + PostgreSQL (Abacus-hosted) · NextAuth v4 · Mapbox GL JS · Tailwind + shadcn/ui.

## Current integrations
- **Mapbox** (maps + geocoding) — env: `MAPBOX_TOKEN`, `NEXT_PUBLIC_MAPBOX_TOKEN`.
- **Overpass / OpenStreetMap** — building polygons + roof area computation.
- **Google SSO** — env: `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`. Google Cloud project is "SolarScout Pro" (project number 1050055939275). OAuth redirect URIs configured for `solarscout-pro-7foyl3.abacusai.app` and the preview host.
- **Google Solar API** — `buildingInsights:findClosest`. Key is `GOOGLE_MAPS_API_KEY` (shared key, works across Maps Platform). Auto-retries with `experiments=EXPANDED_COVERAGE` on 404. Spain/Portugal MEDIUM coverage. Romania/Albania often not indexed.
- **Google Places API (New)** — `places:searchNearby`. Same `GOOGLE_MAPS_API_KEY`. **Enabled in project as of 2026-04-18** — returns real Romanian / Albanian businesses with phone, website, rating, opening hours. The code still uses `Promise.allSettled` so transient 403s fall through safely.
- **Abacus.AI LLM API** (`apps.abacus.ai/v1/chat/completions`) — env: `ABACUSAI_API_KEY`. Model `gpt-5.4-mini` with `response_format: json_schema`. Used exclusively by the outreach-kit endpoint. Never exposed to the client — all calls go server-side through `/api/leads/[id]/outreach`.

## Key files
- `lib/db.ts` — Prisma client singleton.
- `lib/auth.ts` — NextAuth options (Google + credentials).
- `lib/mapbox.ts` — forward geocoding (country-restricted to RO/ES/PT/AL).
- `lib/overpass.ts` — Overpass API commercial-building discovery → returns `OsmBuilding[]` with polygons/roof area.
- `lib/places.ts` — Google Places (New) search, classification, haversine matching. Exports `isPlacesApiConfigured`, `searchNearbyPlacesFanout`, `haversineMeters`.
- `lib/solar.ts` — Solar API client with caching statuses OK/NOT_FOUND/NO_COVERAGE/UNCONFIGURED/ERROR.
- `app/api/leads/generate/route.ts` — Hybrid OSM + Places discovery. Matches OSM buildings to Places by <40m proximity → HYBRID. Unmatched Places become GOOGLE_PLACES (only in OSM-thin markets RO/AL or when OSM found zero).
- `app/api/leads/[id]/solar/route.ts` — Solar data endpoint with 30-day cache.
- `app/api/leads/[id]/outreach/route.ts` — POST-only. Takes `{ language, tone }`, reads the lead + cached solar fields, computes a deterministic business case (see `lib/outreach.ts`), then calls the LLM with a strict JSON schema (`outreach_kit`) to produce `{ email: {subject, body}, call: {opener, keyPoints[], objectionResponse, closing}, whatsapp }`.
- `lib/outreach.ts` — Deterministic economics (system kWp, annual production kWh, self-consumption 70%, €900/kWp install cost, country-specific electricity tariffs ES 0.22 / PT 0.20 / RO 0.18 / AL 0.16 €/kWh, 25-yr lifetime). Also exports `SUPPORTED_LANGUAGES` (en/es/pt/ro/sq) and `OUTREACH_TONES` (professional/friendly/direct) and `defaultLanguageForCountry`.
- `app/leads/[id]/_components/outreach-generator.tsx` — The UI. Gradient violet/amber header, language + tone selects, business-case banner with 4 KPIs, tabs for Email / Call script / WhatsApp. Each tab has editable content, copy buttons, and a direct-action button (mailto:, tel:, wa.me/).

## Data model (Lead)
Core business fields: businessName, businessType, buildingType, address, city, country, latitude, longitude, roofAreaSqm, contactPhone, contactEmail, website, notes, status.

Identity: `osmId` (nullable, unique per user when set), `googlePlaceId` (nullable, app-enforced uniqueness per user).

Data source: `dataSource` = `'OSM' | 'GOOGLE_PLACES' | 'HYBRID'` (default 'OSM').

Google Places extras: `openingHours` (JSON), `googleRating`, `googleRatingCount`.

Solar cache (12 fields): `solarDataFetchedAt`, `solarApiStatus`, `solarImageryQuality`, `solarImageryDate`, `solarMaxPanelCount`, `solarMaxArrayAreaSqm`, `solarMaxSunshineHours`, `solarYearlyEnergyKwh`, `solarCarbonOffsetKgYr`, `solarPanelCapacityWatts`, `solarPanelLifetimeYears`, `solarRawJson`.

Indexes: userId, country, city, businessType, createdAt, solarApiStatus, dataSource, googlePlaceId.

## Behavior & UX rules
- **Never** pull `solarRawJson` in list queries — it can be huge. Always use explicit `select` in the leads list.
- Always serialize Date fields to ISO strings before passing to client components.
- OSM remains primary source for Spain/Portugal (OSM is dense there); Places enriches with contact/rating.
- In Romania/Albania, Places-only leads are allowed since OSM is thin.
- The data source badge (OSM green / Google blue / Hybrid violet) appears on both the leads table and the detail page.
- Places API 403 / disabled just falls through silently via `Promise.allSettled` — never blocks an OSM-only search.
